Hopefully you read up on the gruntfile, if not, get on it lazybones!! Our application uses a grunt plugin called grunt-contrib-watch. This watches all of the files we have bacause I set it up that way! In the gruntfile on lines 24-38 you can see how it is set to watch all of the files pretty much everywhere. You also see that livereload is set to true.

If you already found it in the documentation (Hint Hint) then you know all you have to do is add this line
<script src="//localhost:35729/livereload.js"></script>
to the index.html page located in the layouts folder

Once that is done click on the button

Congrats! You are now reloading live.
Now that you are ready to start developing your application you should look more into Backbone. It gives your code structure that you need. While we are using Hapi to render views and handle routes, Backbone is also able to do all of that. In this stack I am primarily using it for the models. It gives you the opportunity to create objects(models) that can be updated and manipulated. You are able to set and retrieve data based on user interaction.
